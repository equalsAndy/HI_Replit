# Versioning System Documentation

## Overview

The versioning system tracks semantic versions and build numbers for the application. Semantic versions are stored in git; build numbers are derived from git commit count at build time, eliminating merge conflicts when switching branches.

## Core Components

### 1. `version.json` — Central Version Registry (tracked in git)
```json
{
  "version": "2.8.2",
  "description": "Reframe exercise and capability selector enhancements"
}
```

**Purpose**: Single source of truth for the semantic version.
- `version`: Semantic version following major.minor.patch format
- `description`: Human-readable description of the current version

**Not stored here**: Build numbers are derived from `git rev-list --count HEAD` at build time.

### 2. `update-version.sh` — Version Management Script

**Purpose**: Bumps semantic versions and regenerates derived files.

**Usage**:
```bash
./update-version.sh              # Regenerate derived files (no version change)
./update-version.sh patch        # Bug fix: 2.8.2 → 2.8.3
./update-version.sh minor        # New feature: 2.8.2 → 2.9.0
./update-version.sh major        # Breaking change: 2.8.2 → 3.0.0
./update-version.sh 2.9.0        # Set exact version
```

**What it does**:
1. Reads semantic version from `version.json`
2. Derives build number from git commit count
3. Updates `version.json` (only for patch/minor/major bumps)
4. Generates `client/public/version.json` and `public/version.json` (both gitignored)
5. Updates `.env.local` with `VITE_APP_VERSION` and `VITE_BUILD_NUMBER`

### 3. `version-manager.sh` — Environment-Aware Version Script

**Purpose**: Sets version info for specific deployment environments (development, staging, production).

**Usage**:
```bash
./version-manager.sh development patch
./version-manager.sh production
```

### 4. Derived Files (gitignored)

These are generated by the scripts above and consumed at runtime:

| File | Used By | Contains |
|------|---------|----------|
| `client/public/version.json` | Vite dev server | version, build, timestamp, environment |
| `public/version.json` | Production server | version, build, timestamp, environment |
| `.env.local` | Vite build | VITE_APP_VERSION, VITE_BUILD_NUMBER |

## Build Number Strategy

Build numbers come from `git rev-list --count HEAD` — the total commit count on the current branch. This means:
- **No merge conflicts**: Nothing stored in tracked files
- **Branch-aware**: Each branch has its own natural build number
- **Always increasing**: On any given branch, commits only go up
- **Reproducible**: The same commit always produces the same build number

## Frontend Integration

### Environment Variables
```bash
VITE_APP_VERSION=2.8.2      # Semantic version for display
VITE_BUILD_NUMBER=3399      # Git-derived build number
```

### Version Badge Display (`client/src/components/layout/NavBar.tsx`)

| Environment | Detection | Display |
|-------------|-----------|---------|
| Development | `localhost`, port `8080` | `DEV v2.8.2.3399` (red badge) |
| Staging | `app.heliotropeimaginal.com` | `STAGING v2.8.2.3399` (gray badge) |
| Production | Other hostnames | No badge |

## How to Use

### Daily development — no action needed
Build numbers are derived automatically. Just commit and build.

### Releasing a new version
```bash
./update-version.sh minor    # or patch, or major
git add version.json
git commit -m "Bump version to 2.9.0"
```

### Checking current version
```bash
cat version.json                              # Semantic version
git rev-list --count HEAD                     # Current build number
cat public/version.json 2>/dev/null           # Full runtime version info
```

## Troubleshooting

### Version badge not updating
1. Hard refresh browser (`Cmd+Shift+R`)
2. Run `./update-version.sh` to regenerate derived files
3. Restart dev server

### Derived files missing
Run `./update-version.sh` — it regenerates `client/public/version.json`, `public/version.json`, and `.env.local`.

## File Locations

| File | Tracked | Purpose |
|------|---------|---------|
| `version.json` | Yes | Semantic version + description |
| `client/public/version.json` | No (gitignored) | Runtime version for Vite dev |
| `public/version.json` | No (gitignored) | Runtime version for production |
| `.env.local` | No (gitignored) | Vite build variables |
| `update-version.sh` | Yes | Main versioning script |
| `version-manager.sh` | Yes | Environment-aware versioning |
| `create-release.sh` | Yes | Release workflow |
